## SQL review summary

The current SQL is acceptable for a small local dataset, but there are three clear improvements worth making:

- transfer_matcher.py is doing a nested scan over debits and credits, which becomes $O(n^2)$ as imports grow. A join-based query on date, amount, and payment mode would be much faster and simpler to reason about.
- export.py makes several separate count queries for the export summary. Those can be collapsed into one aggregate query so the app does less work per export.
- repository.py currently treats only `NULL` as uncategorized, while the export logic also treats empty strings as uncategorized. That mismatch can leave rows invisible in the review UI. The filter should be `category IS NULL OR category=''`.
